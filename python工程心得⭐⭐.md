# 如何系统、科学的进行一个python工程编程

1. 先做调研，手画出**业务流程图**
2. 使用**excle**整理出具体需要哪些**功能模块**
3. process on画出**系统架构图**
4. **创建工程**，创建需要的模块
5. 编写整个工程的**main文件**，进行各个模块的**逻辑串联**
6. 创建每个模块的**main文件**，进行每个模块的具体的**实现逻辑设计**



几个要点：

1. 前期以跑出来为主，所以清晰的逻辑 > 性能要求
2. 中期进行性能优化，所以 少循环+少落盘 > 清晰的逻辑
3. 并行+大数据量，所以 分布式+并行 > 少循环+少落盘 > 清晰的逻辑
